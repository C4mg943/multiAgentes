# 🧠 Sistema de Agentes - Contexto Global

## Quién soy
Soy estudiante de ingeniería/sistemas en la universidad. Trabajo en proyectos académicos
que van desde apps web completas hasta APIs REST y aplicaciones móviles. Mi nivel es
intermedio-avanzado, aprendo haciendo y necesito código limpio, bien documentado y
que siga buenas prácticas reales de la industria.

## Cómo quiero que me respondas
- Código siempre funcional y completo, no fragmentos a medias
- Explicá brevemente el "por qué" de decisiones importantes
- Si hay algo malo en mi enfoque, decímelo directamente
- Usá español en explicaciones, inglés en el código (variables, funciones, comentarios)
- Cuando hay varias formas de hacer algo, mostrá la mejor y mencioná las otras

## Mi entorno de desarrollo
- OS: Windows / Linux (WSL2)
- Editor: VSCode con extensiones (Prettier, ESLint, GitLens)
- Control de versiones: Git + GitHub
- Gestor de paquetes: npm / pip / maven según el proyecto

## Stack principal por proyecto
| Tipo de proyecto | Stack |
|---|---|
| Frontend Web | React 18 + Next.js 14 + Tailwind CSS |
| Frontend Empresarial | Angular 17 + TypeScript |
| Backend JS | Node.js + Express + Prisma |
| Backend Python | FastAPI + SQLAlchemy |
| Backend Java | Spring Boot 3 |
| Base de datos | PostgreSQL (principal), MongoDB (cuando hay docs) |

## Reglas generales de código
1. Siempre tipado: TypeScript en JS/TS, type hints en Python
2. Variables y funciones en camelCase (JS/TS), snake_case (Python)
3. Clases en PascalCase en todos los lenguajes
4. Archivos de componentes: PascalCase.tsx, utilidades: camelCase.ts
5. Sin console.log en producción, usar logger apropiado
6. Manejo de errores explícito, nunca swallow exceptions
7. Commits en inglés, formato: `type(scope): description`
